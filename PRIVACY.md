## 1. Who We Are

Vouqis ("we", "us", "our") provides the Vouqis AI Agent Reliability Gateway — a suite of tools that sit between AI agents and MCP servers to validate, rate-limit, retry, and audit every request. The suite consists of:

- **Vouqis CLI** — a local command-line proxy installed on your machine
- **Vouqis Dashboard** — a web application for reliability reports and Pro subscription management

This policy covers both surfaces. Where practices differ between them, we say so explicitly.

## 2. What Data We Collect and Why

### 2.1 CLI — Anonymous Analytics

**Applies to**: anyone running the `vouqis` CLI command

When you run Vouqis and `POSTHOG_API_KEY` is set in the environment, the CLI sends anonymised usage events to PostHog. These events contain:

| Event | Data sent |
|---|---|
| `logs_command_run` | Number of audit-log entries, whether `--summary` was used |
| `config_loaded` | Filesystem path of config file, number of upstreams |
| `proxy_config_error` | Config file path, error message text |
| `proxy_start_error` | Listen address, error message text |
| `proxy_started` | Upstream **hostname only** (never the full URL), listen address, timeout/retry/rate-limit settings, policy flags |
| `proxy_stopped` | Upstream hostname, session uptime in milliseconds |
| `request_allowed / blocked / retried / rewritten` | MCP method name, MCP tool name, latency, upstream hostname, block reason (a static validator string — never request content) |
| Exception events | MCP method name, tool name, upstream hostname, error type and stack |

**What is never included in analytics**: request or response content, full upstream URLs, authentication tokens, names, or email addresses.

Each analytics session is tagged with a **random UUID** stored at `~/.vouqis/analytics-id`. This UUID is generated locally and is not linked to any personal identifier.

**If `POSTHOG_API_KEY` is absent or empty**, the PostHog SDK is entirely disabled — no events are sent and no network requests are made.

**Purpose**: understand feature adoption, detect recurring errors, improve reliability.  
**Legal basis (GDPR)**: Legitimate interests (Art. 6(1)(f)).

### 2.2 Dashboard — Account and Subscription Data

**Applies to**: users who visit the Vouqis web dashboard and purchase a Pro subscription

When you purchase Vouqis Pro, we collect:

| Data | Purpose |
|---|---|
| **Email address** | Deliver your API key, send your receipt, identify your subscription |
| **Razorpay payment ID and order ID** | Verify your payment, link your subscription record, generate your receipt |
| **Vouqis API key** (generated by us, 64-char hex) | Authenticate your Pro tier usage in the reports API |
| **Subscription status and plan** | Determine your access level (free vs. Pro, report retention days) |

We do **not** receive your credit card number, CVV, UPI PIN, or bank credentials — those are entered directly into Razorpay's secure checkout and never touch our servers.

**Purpose**: fulfill your subscription, deliver the service you paid for, provide customer support.  
**Legal basis (GDPR)**: Performance of a contract (Art. 6(1)(b)).

### 2.3 Dashboard — MCP Reliability Reports

**Applies to**: users who submit MCP server reliability reports via the API

When you run a reliability probe and submit results, we store:

| Data | Purpose |
|---|---|
| **MCP server URL** | Identify the server being tested |
| **Trust score, verdict, pass/fail counts, latency** | Generate and display the reliability report |
| **Top failure categories and probe results** | Provide actionable detail in the report |

Reports submitted without a Pro API key expire after 7 days. Pro reports are retained for 90 days, then deleted.

**Purpose**: provide the reliability reporting service.  
**Legal basis (GDPR)**: Legitimate interests (Art. 6(1)(f)) for free-tier reports; performance of contract (Art. 6(1)(b)) for Pro-tier reports.

- Sensitive categories of personal data (health, biometric, political, religious)

## 4. Cookie Policy

### CLI

The Vouqis CLI does not use cookies. It runs outside any browser environment and cannot set or read cookies.

### Dashboard (web application)

The Vouqis web dashboard uses the following cookies:

| Cookie | Type | Purpose | Duration |
|---|---|---|---|
| Supabase auth session token | Strictly necessary | Maintain your authenticated session if you sign into the dashboard | Session / 1 week |
| `vouqis-theme` | Functional | Remember your dark/light mode preference | 1 year |

We do **not** currently use advertising cookies, third-party tracking pixels, or analytics cookies on the dashboard beyond what PostHog sets (see below).

**PostHog web SDK** (if loaded on the dashboard): PostHog sets a first-party cookie (`ph_*`) to identify your browser session across page loads, linked to an anonymous ID — not to your email. You can opt out of PostHog's web tracking by enabling "Do Not Track" in your browser settings.

**No consent banner is required for strictly necessary cookies.** If we add optional cookies in the future, we will implement a consent banner in compliance with PECR and ePrivacy regulations before doing so.

## 7. Payment Processing

Vouqis Pro subscriptions are processed via **Razorpay**. When you complete a payment:

1. You enter your payment credentials (UPI, card, netbanking) directly into Razorpay's secure checkout — this data goes to Razorpay and never to our servers.
2. Razorpay returns a `payment_id`, `order_id`, and cryptographic `signature` to your browser.
3. Your browser sends those three values, plus your email address, to our verification endpoint.
4. We verify the signature using HMAC-SHA256 against our Razorpay secret key, provision your subscription in Supabase, and email you your API key.

We do not store your card number, UPI VPA, bank credentials, or CVV at any point. For payment disputes or refunds, contact hello@vouqis.tech. Razorpay's privacy policy: https://razorpay.com/privacy/

## 9. Data Security

- All data transmitted between your machine and our services uses TLS 1.2 or higher
- Razorpay payment verification uses HMAC-SHA256 signature validation — we reject any unverified payment notification
- Vouqis API keys are generated with 32 bytes of cryptographically secure random data (`crypto.randomBytes(32)`)
- Supabase access uses a service-role key stored as an environment variable, not embedded in client-side code
- The CLI never transmits MCP request or response content to any external service

## 10. How to Control Your Data

### Disable CLI analytics entirely
Do not set `POSTHOG_API_KEY` in your environment. With no key present, the PostHog SDK disables itself; no events are sent, no listeners are registered.

### Reset your anonymous analytics ID
```bash
rm ~/.vouqis/analytics-id
```
A new UUID is generated on the next command run. This severs continuity without stopping analytics.

### Delete your local audit log
```bash
rm ./vouqis-audit.log
```
Or redirect it during proxy startup: `--log-file /dev/null`

### Request deletion of your subscription data
Email hello@vouqis.tech with "Erasure Request" in the subject line. Note that we may retain minimum records required for accounting obligations.
